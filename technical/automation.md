|------------|----------|-----------|----------------|---------|

## 2. P1 — 국내음반 신보 이메일 파싱 자동화

### 목표
공급사(M2M, 포스온 등) → 신보 안내 이메일 수신 → 플랫폼 상품 자동 등록 → 거래처 알림 발송

### 구현 방식

```
[공급사 이메일 발송]
    │
    ▼ Gmail API 또는 IMAP 폴링 (5~10분 간격)
[이메일 수신 감지]
    │
    ▼ 첨부파일(엑셀/PDF) 또는 본문 텍스트 추출
[원문 데이터 추출]
    │
    ▼ GPT API (구조화 추출 프롬프트)
[상품 정보 파싱]
 - 아티스트명
 - 앨범명 / 버전명
 - JAN 코드
 - 발매일 / 주문 마감일
 - 정가 / 공급가
 - 판매 조건 (랜덤/세트 여부)
 - 포함 구성품
    │
    ▼ 중복 체크 (JAN 코드 기준)
[플랫폼 DB 등록 — draft 상태]
    │
    ▼ 어드민 검수 알림
[담당자 검수 → 승인]
    │
    ▼ published 상태 전환
[거래처 알림 발송]
```

### 참고: 손정욱 대표 작성 프롬프트
- 파일명: `250916_메일+DB(국내음반)_프롬프트.txt` (이메일 공유됨)
- 이미 담당자가 GPT 활용 경험 보유 → 프롬프트 템플릿 기반으로 시작 가능

### 이메일 유형별 처리 전략
| 이메일 유형 | 처리 방식 |
|------------|---------|
| 신보 안내서 (일반) | 자동 파싱 → draft 등록 |
| 신보 추가 업데이트 | 기존 상품 찾아서 정보 업데이트 |
| 재입고 안내 | 해당 상품 재고 상태 업데이트 |
| 구보 전환 안내 | 상품 status 변경 (NEW_RELEASE → IN_STOCK) |
| 마감 안내 | 주문 마감일 업데이트 + 거래처 알림 |

## 3. P2 — 일본 도서·잡지 신보 파싱

### 특징
- 일본어 엑셀 첨부파일 형식
- 마감일(締切), 상품명, 가격, JAN 코드 포함
- 웬즈데이가 원본에서 관심 상품 선별 후 자체 배포 이메일 작성

### 자동화 방향
1. 일본 공급사 이메일 수신 감지
2. 일본어 엑셀 파일 파싱 (GPT API, 일본어 지원)
3. 전체 상품을 draft로 등록
4. 담당자가 어드민에서 배포 대상 상품 선택
5. 선택된 상품에 대해 거래처 알림 자동 발송

---

## 4. P2 — 패킹리스트 자동 처리

### 현재 문제
- 같은 품목이 박스 단위로 중복 나열 → 수동 합산
- JAN 코드 → 품번 매핑 필요
- 기존에는 GPT로 수동 처리 (수 시간 → 수십 분으로 단축됨)

### 자동화 방향
1. 패킹리스트 파일 업로드 (어드민 기능)
2. GPT API로 품목별 합산 자동 처리
3. JAN 코드 → 품번 자동 매핑 (DB 기반 룩업)
4. 처리 결과를 입고 현황에 자동 반영

---

## 5. P2 — 거래처별 맞춤 알림

### 현재 상태
- 모든 거래처에 동일한 신보 안내 발송
- 거래처마다 취급 품목이 다름 (국내반 전문 / 굿즈 전문 / 일본 상품 전문 등)

### 자동화 방향
1. 거래처 프로필에 관심 카테고리 및 아티스트 설정
2. 신보 등록 시 매칭되는 거래처에만 선별 알림 발송
3. 과거 주문 이력 분석 → 관심 패턴 자동 추천 (장기적)

---

## 6. P3 — Nippan IPS 플랫폼 연동

### 기회
- Nippan IPS(ts-base.jp) 플랫폼에 올라오는 상품과 재고를 연동하면 일본 재고 판매 채널 확보 가능
- Nippan 측이 전세계 바이어에게 열어둔 플랫폼 → Koreal을 통해 중개 가능

### 구현 방식 (검토 필요)
- 정기적 상품 목록 크롤링 (robots.txt 확인 필수)
- 상품 정보를 Koreal 카탈로그로 동기화
- 주문 발생 시 ts-base.jp에 자동 또는 반자동 발주

### 주의사항
- 계정 공유를 매우 엄격하게 관리하는 플랫폼
- 주문 후 취소 어려움 → 테스트 단계에서 담당자와 반드시 협의 후 진행
- 크롤링 전 법적·계약적 허용 여부 확인 필요

## 7. 카카오톡 기반 주문 처리 (예외 케이스)

### 현재 상황
- 일부 공급사(사운드웨이브 등)는 카카오톡으로 이미지(스크린샷) 형태로 상품 안내
- 담당자가 스크린샷 캡처 → GPT에 이미지 전달 → 텍스트 추출

### 단기 대안
- 카카오톡 수신 → 특정 이메일로 포워딩 → 이메일 파싱 파이프라인에 편입
- 어드민에 이미지 업로드 → GPT Vision API 파싱 기능 제공

### 장기 방향
- 공급사에게 플랫폼 직접 등록 또는 API 제공 요청

## 8. 자동화 구현 체크리스트

### Phase 1 (MVP) — 현재 범위

**이메일 파싱·자동 수집은 구현하지 않는다.** 상품은 [시스템 아키텍처](./architecture.md)대로 **관리자가 Next 어드민에서 수동 등록**한다.

- [ ] (해당 없음) 이메일 파싱 — **후속 Phase 2 이하**

### Phase 2
- [ ] 이메일 수신 서버 설정 (신보 안내 전용 이메일 도메인)
- [ ] GPT API 기반 신보 이메일 파싱 (국내음반)
- [ ] 파싱 결과 어드민 검수 UI
- [ ] 검수 완료 후 상품 자동 등록
- [ ] 거래처 이메일/알림톡 발송
- [ ] 일본 도서·잡지 이메일 파싱 (일본어)
- [ ] 패킹리스트 자동 처리 모듈
- [ ] 거래처별 관심 카테고리 기반 맞춤 알림
- [ ] 이메일 유형 자동 분류 (신보/재입고/마감 등)

### Phase 3
- [ ] Nippan IPS 연동 (검토 후 결정)
