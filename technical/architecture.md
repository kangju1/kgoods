# 시스템 아키텍처

## 1. 플랫폼 구성 개요

Koreal 플랫폼은 크게 세 가지 레이어로 구성된다:

```
┌─────────────────────────────────────────────────────────┐
│                      외부 데이터 소스                      │
│  공급사 이메일  |  Nippan IPS 플랫폼  |  카카오톡  |  ERP  │
└───────────────┬──────────────────────────────────────────┘
                │ 자동 수집 / 파싱
┌───────────────▼──────────────────────────────────────────┐
│                    Koreal 백엔드                           │
│  상품 관리  |  주문 관리  |  거래처 관리  |  알림  |  정산  │
└───────────────┬──────────────────────────────────────────┘
                │
┌───────────────▼──────────────────────────────────────────┐
│                    Koreal 프론트엔드                        │
│         바이어 포털 (주문)  |  운영자 어드민 (관리)          │
└─────────────────────────────────────────────────────────┘
```

---

## 2. 주요 모듈

### 2-1. 상품 카탈로그 모듈

- 상품 등록·수정·삭제 (어드민)
- 카테고리 트리 관리 (대분류: 국내반/수입반/MD 등)
- 아티스트 마스터 관리
- 거래처별 가격 그룹 (프라이스 티어) 적용
- 재고 수량 관리
- 신보·구보·품절 상태 관리

### 2-2. 신보 안내 자동화 모듈

- 공급사 이메일 수신 파싱 (이메일 훅 또는 폴링)
- AI 기반 상품 정보 추출 (상품명, 아티스트, 발매일, 가격, JAN 코드 등)
- 플랫폼 상품 자동 등록 (draft 상태로 올린 후 담당자 검수)
- 등록 완료 시 알림톡/이메일 자동 발송
- 거래처별 관심 상품 필터링 (과거 구매 이력 기반)

### 2-3. 주문 관리 모듈

- 거래처 로그인 후 장바구니 → 주문
- 신보(예약) vs 구보(즉시 출고) 주문 흐름 분리
- 수주서 자동 발행
- 주문 상태 추적 (접수→확정→출고→완료)
- 재고 확인 후 주문 확정 워크플로 (구보 상품)

### 2-4. 거래처 관리 모듈

- 거래처 등록 및 심사 (B2B 특성상 가입 승인 방식)
- 거래처별 공급가 그룹 설정
- 결제 방식 지정 (선결제 / 후불 정산)
- 거래처별 취급 카테고리 설정 (알림 필터링 연동)

### 2-5. 알림 모듈

- 신보 등록 시 거래처 알림톡/이메일 자동 발송
- 재입고 알림
- 주문 상태 변경 알림 (확정, 출고 등)
- 마감 임박 주문 독려 알림

### 2-6. 어드민 (운영자 도구)

- 신보 파싱 결과 검수 및 수정
- 수동 상품 등록
- 주문 현황 대시보드
- 입출고 현황 관리
- 거래처 관리

---

## 3. 외부 시스템 연동

| 시스템 | 연동 방식 | 목적 |
|--------|----------|------|
| 공급사 이메일 | 이메일 훅 (Gmail API / IMAP 폴링) | 신보 안내 자동 수집 |
| Nippan IPS (ts-base.jp) | 웹 크롤링 (주의: 계정 공유 제한 있음) | 일본 재고 상품 자동 동기화 |
| 카카오 알림톡 | 카카오 비즈니스 API | 거래처 주문 알림 발송 |
| ERP (이카운트) | API 연동 또는 수동 동기화 | 수주서·거래명세서 발행 |
| 결제 게이트웨이 | PG사 API (토스페이먼츠 등) | 선결제 처리 |

---

## 4. 기술 스택 고려사항

### 백엔드
- RESTful API 또는 GraphQL
- 이메일 파싱: AI(GPT API) 활용 구조화 데이터 추출
- 작업 큐: 이메일 수신 → 파싱 → 등록 비동기 처리

### 프론트엔드
- 바이어 포털: 반응형 웹 (모바일 포함)
- 어드민: 데스크탑 중심 운영 도구
- 다국어 지원 (한국어, 영어, 일본어) — 글로벌 바이어 대상

### 인프라
- 클라우드 기반 배포 (AWS / GCP)
- 이메일 수신 전용 도메인 설정 (신보 안내 포워딩용)

## 5. 데이터 흐름

```
[공급사 신보 안내 이메일]
    │
    ▼ (Gmail API / IMAP)
[이메일 수신 서버]
    │
    ▼ (GPT API 파싱)
[상품 정보 추출: 상품명, JAN, 발매일, 가격...]
    │
    ▼
[플랫폼 상품 DB (draft 상태)]
    │
    ▼ (담당자 검수 → 승인)
[플랫폼 상품 DB (published 상태)]
    │
    ▼ (알림 모듈)
[거래처 알림톡 / 이메일 발송]
    │
    ▼ (거래처 로그인 → 주문)
[주문 DB → 수주서 발행]
    │
    ▼ (재고 확인 → 출고)
[입출고 현황 업데이트]
    │
    ▼ (출고 완료)
[거래명세서 → 정산]
```
