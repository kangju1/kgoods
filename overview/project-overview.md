# 프로젝트 개요

## 1. 배경 및 목적

### 웬즈데이 (Wednesday)

웬즈데이는 국내 및 일본의 음반·잡지·굿즈(MD) 등 K-콘텐츠 물리 상품을 B2B로 도매 유통하는 회사다. 파주시 문발로 127(웬즈데이빌딩)에 본사 및 물류 거점을 두고 있으며, 약 4년 이상의 상품 취급 이력과 다수의 매입·매출 거래처 네트워크를 보유하고 있다.

현재 업무는 이메일·카카오톡·엑셀·ERP(이카운트)를 조합하여 수동으로 운영되고 있으며, 신보 안내 수신부터 거래처 발주 접수, 출고, 정산에 이르는 전 과정이 담당자의 수작업에 크게 의존하고 있다.

### 프로젝트 목표

웬즈데이의 검증된 소싱 역량과 거래처 네트워크를 기반으로, **디지털 B2B 도매 플랫폼(Koreal)**을 구축한다.

- **디지털화**: 이메일·엑셀·카카오톡에 분산된 업무를 단일 플랫폼으로 통합
- **자동화**: 신보 안내 수신 → 거래처 배포 → 주문 접수 → 수주서 발행의 반복 작업 자동화
- **글로벌화**: 한국·일본에 한정된 거래처 기반을 글로벌 바이어로 확장

## 3. 핵심 가치 제안

### 3-1. 거래처(바이어) 관점

| 현재 | 플랫폼 구축 후 |
|------|--------------|
| 이메일·카카오톡으로 신보 정보 수신 | 플랫폼에서 전체 카탈로그 실시간 탐색 |
| 엑셀 발주서 작성 후 이메일/카톡 회신 | 플랫폼 내 장바구니 → 주문 원클릭 처리 |
| 거래처마다 다른 가격 체계로 혼란 | 거래처별 공급가 자동 적용 |
| 신보 정보 누락·지연 빈번 | 신보 등록 즉시 알림톡/이메일 자동 발송 |
| 재고 파악 어려움 | 실시간 재고 현황 확인 가능 |

### 3-2. 웬즈데이(운영자) 관점

| 현재 | 플랫폼 구축 후 |
|------|--------------|
| 신보 안내 이메일 수동 작성·발송 | 신보 등록 → 자동 메일/알림톡 발송 |
| 거래처마다 다른 발주서 양식 수동 처리 | 단일 주문 인터페이스로 표준화 |

## 6. 프로젝트 범위 (1차)

- 상품 카탈로그 관리 (국내반 중심 → 전 카테고리 확장)
- 신보 안내 자동화 (이메일 파싱 → 플랫폼 등록 → 거래처 알림)
- 거래처 관리 및 로그인 기반 가격 차등 공개
- 주문 접수 및 수주서 발행 자동화
- 재고 현황 관리

> 일본 수입 물류(통관, B/L 처리 등) 자동화는 복잡성으로 인해 2차 범위로 분류한다.
