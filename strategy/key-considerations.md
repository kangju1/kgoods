# 핵심 고려사항 및 전략적 제언

## 1. 비즈니스 복잡도 — 주의해야 할 영역

### 1-1. 가격 체계의 이중성

K-POP 도매 가격은 두 가지 체계가 혼재한다:
- **도서·잡지**: 정가 × 공급율 (규칙적, 자동 계산 가능)
- **음반**: 공장출고가 기준 (공급율 적용 불가, 거래처별 개별 가격 필요)

> **전략적 제언**: 플랫폼 초기부터 '가격 그룹(프라이스 티어)' 구조를 설계해야 한다. 나중에 추가하면 데이터 마이그레이션 비용이 크다. 음반은 거래처 × 상품 단위로 개별 가격이 필요하다는 점을 데이터 모델에서 허용해야 한다.

### 1-2. 판매 조건의 유동성

유통사가 사전 공지 없이 "랜덤만 판매", "세트 판매 불가" 등 조건을 변경한다. 이 변경이 거래처에 제때 전달되지 않으면 주문 혼란과 분쟁이 발생한다.

> **전략적 제언**: 상품별 판매 조건을 구조화된 속성으로 관리하고, 조건 변경 시 자동 알림 기능을 구현한다. 조건 변경 이력도 보존해야 한다.

### 1-3. 단독/한정 유통 상품의 특수성

사운드웨이브 단독 MD 등 일부 상품은 "해외만 판매", "특정 거래처만 주문 가능" 등 유통 제한이 있다.

> **전략적 제언**: 상품별 가시성(visibility) 제어 기능이 필요하다. 특정 거래처 그룹에만 노출되는 상품 설정 기능을 구현해야 한다.

## 2. 운영 리스크

### 2-1. 신보 파싱 정확도

GPT 기반 파싱은 완벽하지 않다. 특히:
- 이메일 형식이 공급사마다 상이
- 동일 공급사도 이메일 양식을 수시로 변경
- 이미지(스크린샷) 기반 안내는 OCR 정확도 이슈

> **전략적 제언**: 자동 파싱 → 반드시 담당자 검수 단계를 거친 후 상품 등록. "draft 승인" 워크플로를 MVP부터 포함해야 한다. 파싱 실패 케이스를 별도 대기열에 모아 담당자가 수동 처리할 수 있어야 한다.

### 2-2. 이메일 채널 의존도

현재 웬즈데이의 정보 수신 채널이 이메일에 크게 의존한다. 공급사가 이메일 대신 자체 플랫폼(예: 워크시트)으로 전환할 경우 파이프라인이 단절된다.

> **전략적 제언**: 이메일 외에 수동 등록 어드민 UI를 항상 병행 제공한다. 카카오톡 이미지 기반 정보는 이미지 업로드 → GPT Vision 파싱 경로로 처리한다.

### 2-3. 재고 정보의 실시간성

신보(예약)는 재고 이슈가 없지만, 구보(상시재고)는 실제 창고 재고와 플랫폼 재고가 실시간 동기화되어야 한다. 동기화 지연 시 과주문 → 취소 → 거래처 신뢰도 하락.

> **전략적 제언**: 창고 입출고와 연동된 재고 관리 모듈을 초기부터 설계한다. 적어도 수동 재고 업데이트를 쉽게 할 수 있는 어드민 UI가 필요하다.

### 2-4. 거래처 온보딩

B2B 플랫폼의 특성상 거래처가 기존 방식(이메일·카카오톡)에서 새 플랫폼으로 전환하는 데 저항이 있을 수 있다.

> **전략적 제언**: 초기에는 기존 이메일 방식도 병행 유지한다. 플랫폼 주문이 더 편리하다는 것을 체감하게 만들어야 전환이 자연스럽게 일어난다. 특히 주문 히스토리 조회, 재고 현황 실시간 확인 같은 기능이 온보딩 유인이 된다.

## 3. 기술적 고려사항

### 3-1. 다국어 지원은 처음부터

글로벌 바이어를 타겟으로 하는 플랫폼이므로, 나중에 다국어를 추가하면 대규모 리팩토링이 필요하다.

> **전략적 제언**: i18n(국제화) 구조를 초기 아키텍처에 포함한다. 최소 한국어·영어부터 시작하고, 일본어는 Phase 2에 추가한다.

### 3-2. Nippan IPS 크롤링 리스크

계정 공유에 민감한 플랫폼이며, 자동화된 접속 감지 시 계정 차단 가능성이 있다.

> **전략적 제언**: Nippan IPS 측과 공식 API 또는 데이터 피드 제공 협의를 먼저 시도한다. 크롤링은 법적·계약적 검토 후 최후 수단으로 사용한다.

### 3-3. ERP 연동 vs. 자체 수주 관리

이카운트 ERP와의 연동은 구현 복잡도가 높다. API 제공 여부 및 비용 확인이 필요하다.

> **전략적 제언**: MVP 단계에서는 플랫폼 자체 수주서 발행 기능을 구현하고, ERP는 수동으로 입력하는 병행 운영으로 시작한다. 주문 볼륨이 커지면 ERP 자동 연동을 고려한다.

## 4. 전략적 기회

### 4-1. 배포 누락 상품의 매출화

현재 담당자 여력 부족으로 약 30~40%의 신보가 거래처에 배포되지 못하고 있다. 플랫폼 자동화로 100% 배포 시 매출 기회가 직접 증가한다.

### 4-2. 그라비아·화보집 카테고리

- 국내 교보·예스24와 해외 아마존에서 확인되는 꾸준한 수요
- 거래처가 메신저나 이메일로 주문하기 민망한 상품 → 플랫폼 익명 주문 환경에서 오히려 수요 증가 기대
- 어떤 경쟁 B2B 플랫폼도 이 카테고리를 체계적으로 다루지 않음

> **전략적 제언**: 성인 콘텐츠 전용 섹션을 별도로 구성하고, 거래처 계정 인증 후 접근 가능하도록 설계한다.

### 4-3. 아티스트 인기도 기반 알림 타게팅

과거 주문 이력 + 차트 데이터를 결합하면 "이 거래처는 어떤 아티스트 상품을 주문할 가능성이 높은가"를 예측할 수 있다.

> **전략적 제언**: 초기에는 거래처가 직접 관심 아티스트/카테고리를 설정하는 방식으로 시작한다. 데이터가 쌓이면 AI 추천으로 발전시킨다.

### 4-4. 팔릴 만한 상품 선별 자동화

손정욱 대표가 직접 "판매할만한 상품"을 선별하는 작업이 현재 병목이다. 이를 다음 신호로 자동화할 수 있다:
- 아티스트 차트 순위 (하이토, 써클 차트)
- 과거 동일 아티스트 상품 판매 실적
- 거래처별 관심 카테고리 매칭

플랫폼 개발 시작 전에 정리되어야 할 마스터 데이터:

| 항목 | 현재 상태 | 필요 작업 |
|------|----------|---------|
| 거래처 마스터 | ERP + 담당자 기억 분산 | 공식 거래처 목록 엑셀 정리 |
| 상품 마스터 | 4년치 입출고현황 엑셀 | 카테고리 기준 정규화 |
| 공급사 이메일 도메인 목록 | 불완전 (60~70% 수신 중) | 전체 공급사 이메일 목록 확보 |
| 거래처별 공급가 체계 | 담당자 기억 | 거래처 × 상품 유형별 가격 정리 |
| JAN 코드 ↔ 품번 매핑 | 부분적 엑셀 | 완전한 매핑 테이블 구축 |
