# 개발 로드맵

## 전제: 데이터 준비 (개발 착수 전)

플랫폼 개발에 앞서 마스터 데이터 정비가 필요하다.

| 작업 | 담당 | 기한 |
|------|------|------|
| 거래처 마스터 정리 (코드, 정식명, 담당자, 결제유형) | 손정욱 대표 | 개발 착수 전 |
| 공급사 이메일 전체 목록 정리 | 손정욱 대표 | 개발 착수 전 |
| 거래처별 공급가 그룹 정의 | 손정욱 대표 + 강주원 | 개발 착수 전 |
| 기존 상품 DB 카테고리 정규화 (샘플) | 강주원 | 개발 착수 전 |

---

## Phase 1 — MVP (핵심 자동화 + 기본 주문 플로우)

**목표**: 신보 이메일 자동 파싱 → 플랫폼 등록 → 거래처 주문 접수 자동화

### 기능 목록

**어드민 (운영자)**
- [ ] 상품 수동 등록 / 편집 / 삭제
- [ ] 카테고리 관리 (국내반/수입반/MD 등)
- [ ] 아티스트 마스터 관리
- [ ] 거래처 등록 및 가격 그룹 설정
- [ ] 신보 이메일 파싱 파이프라인 (국내음반)
  - 이메일 수신 → GPT 파싱 → draft 목록
  - draft 검수 및 승인 UI
- [ ] 주문 현황 대시보드 (접수 / 확정 / 출고)
- [ ] 재고 수동 업데이트

**바이어 포털**
- [ ] 거래처 로그인 (승인 기반 가입)
- [ ] 상품 카탈로그 탐색 (카테고리 / 아티스트 / 검색)
- [ ] 거래처별 공급가 표시 (로그인 후 가격 공개)
- [ ] 장바구니 → 주문 접수
- [ ] 주문 내역 조회

**알림**
- [ ] 신보 등록 시 이메일 자동 발송 (전체 거래처)
- [ ] 주문 접수 확인 알림 (거래처)
- [ ] 주문 확정 / 출고 상태 알림 (거래처)

## Phase 2 — 자동화 고도화 + 글로벌 대응

**목표**: 전 카테고리 신보 자동화, 거래처별 맞춤 알림, 다국어 지원

### 기능 목록

**자동화 확장**
- [ ] 일본 도서·잡지 신보 이메일 파싱 (일본어)
- [ ] 패킹리스트 자동 합산 처리 모듈
- [ ] JAN 코드 ↔ 품번 자동 매핑
- [ ] 이메일 유형 자동 분류 (신보/재입고/구보전환/마감)
- [ ] 카카오톡 이미지 → 어드민 이미지 업로드 파싱

**맞춤화**
- [ ] 거래처별 관심 카테고리/아티스트 설정
- [ ] 관심 설정 기반 맞춤 신보 알림
- [ ] 카카오 알림톡 발송 연동 (이메일 외 채널 추가)

**글로벌 대응**
- [ ] 영어 다국어 지원 (i18n)
- [ ] 해외 결제 수단 (카드, 페이팔 등)
- [ ] 해외 거래처 온보딩 프로세스

**운영 도구**
- [ ] 재고 현황 대시보드 (미입고 / 과입고 파악)
- [ ] 거래처별 주문 실적 리포트
- [ ] 아티스트별 판매 실적 분석

## Phase 3 — 생태계 확장

**목표**: 외부 플랫폼 연동, AI 추천, 거래처 자가 운영

### 기능 목록

- [ ] Nippan IPS 재고 연동 (협의 또는 크롤링)
- [ ] ERP(이카운트) 연동 자동화 (수주서 자동 등록)
- [ ] 아티스트 차트 데이터 연동 (하이토/써클 차트)
- [ ] AI 기반 관심 상품 추천 (주문 이력 학습)
- [ ] 일본어 다국어 지원
- [ ] 공급사 셀프 상품 등록 포털 (공급사 계정)
- [ ] 성인 콘텐츠 전용 섹션 (인증 거래처 한정)

## 파일럿 운영 전략

Phase 1 완료 후 바로 전체 거래처에 오픈하지 않고, 2~3곳의 협조적인 거래처와 파일럿 운영을 권장한다.

**파일럿 거래처 선정 기준**
- 현재도 이메일 발주를 규칙적으로 사용하는 거래처
- 담당자가 디지털 툴에 익숙한 거래처
- 다양한 카테고리를 주문하는 거래처 (기능 검증 범위 확대)

**파일럿 기간 목표**
- 주문 플로우 실사용 검증
- 신보 파싱 정확도 검증
- 가격 체계 오류 발견 및 수정
- 거래처 피드백 수렴 → 개선
