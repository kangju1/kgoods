# 핵심 업무 프로세스

## 플랫폼 잠재고객(신규 바이어) 사용자 여정 — 1차 가설

**전제**: 1차 타겟은 기존에 웬즈데이와 계약·수기 관리 중인 거래처만이 아니라, **중·소 규모 잠재 바이어**다. **획득**은 구글 등 **검색**을 통한 상품 페이지 노출·유입을 핵심 경로로 가정한다.

| 단계 | 설명 | 시스템/운영 |
|------|------|-------------|
| 1 | 신보 등 소식을 듣고 해당 상품을 구매하기 위해 검색 포털에 검색한다. (예: `OO 앨범`) | SEO/SEM, 콘텐츠·메타데이터 |
| 2 | 검색 결과에 **우리 사이트(상품 관련 페이지)**가 노출된다 | 인덱싱·랜딩 품질 |
| 3 | 클릭하여 **상품 상세**로 유입한다. (첫 방문, 비로그인 가능) | 공개 PDP |
| 4 | 구매(주문)를 위해 **이메일 인증 기반 가입**이 필요하다. **전화번호는 선택** 입력. | 인증·계정 |
| 5 | 가입 후 **주문을 접수**한다. (상품·수량) | 카트·주문 API/화면 |
| 6 | 접수된 주문은 **관리자(어드민) 사이트**에서 확인된다 | 어드민 주문 목록·상세 |
| 7 | 이후 **견적·세부 발주·확정** 등은 **우선 수기**로 진행한다(전화·메일·기존 ERP 등). | 운영 프로세스 문서화 |

> 위 7단계는 **1차 제품 범위와 운영 분업**을 정의한다. 내부적으로는 아래 **기존 8단계 B2B 운영**과 연결되며, 7번 이후 단계의 자동화는 로드맵 후속 페이즈에서 다룬다.

---

## 기존 B2B 운영: 전체 플로우 개요

웬즈데이의 B2B 도매 업무는 크게 8단계로 구성된다. 각 단계는 현재 수동으로 처리되고 있으며, 플랫폼 구축 시 자동화 가능 여부를 함께 표시한다.

```
[1] 신보안내 입수
      ↓
[2] 신보안내 배포
      ↓
[3] 주문 접수 → (수주서 발행) → [3-5] 결제 (신규 거래처 / 플랫폼 주문)
      ↓
[4] 발주 (발주서 발행)
      ↓
[5] 구매 (매입)
      ↓
[6] 판매 (거래명세서 발행)
      ↓
[7] 출고
      ↓
[8] 정산
```

---

## 단계별 상세

### [1] 신보안내 입수

**현재 방식**
- 유통대행사(포스온, M2M 등)·음반기획사·일본 공급사 등에서 이메일로 신보 안내서 발송
- 일부는 카카오톡 메시지 또는 이미지(스크린샷)로 수신
- 이메일 발신처가 다수(유통사 1곳에서도 여러 이메일 주소 발송)
- 패킹리스트는 박스 단위로 중복 나열되어 품목별 합산 필요

**신보 안내 유형 (5종)**
| 유형 | 설명 |
|------|------|
| 신보 안내서 | 신규 출시 상품 정보 |
| 신보 추가 업데이트 | 기존 안내서에 정보 추가 |
| 재입고 안내 | 품절 후 재고 확보 안내 |
| 구보 전환 안내 | 신보 → 구보(상시재고) 전환 |
| 주문수량 회신 안내 | 마감 임박 주문 독려 |

**특수 케이스**
- **사운드웨이브 단독 유통 상품**: 메일 또는 카톡 이미지로 수신. 정해진 발송 채널 없음
- **일본 그라비아·화보집**: 수위 있는 콘텐츠이나 수요 있음. 별도 카테고리 필요
- **단독/한정 유통 조건부 상품**: "해외만 판매", "한국 유통 불가" 등 조건 수반

**자동화 우선순위**: 국내음반 신보안내 이메일 파싱 → 전체 카테고리 확장

---

### [2] 신보안내 배포

**현재 방식**
- 수신한 신보 안내서 중 거래처에서 관심 가질 상품을 수동으로 선별
- 자체 양식의 배포 이메일로 작성하여 거래처에 발송
- 담당자 여력 부족으로 전체 신보 중 일부(약 60~70%)만 배포

**문제점**
- 담당자 병목: 하루 10~100통 이상의 신보 안내 이메일 중 선별 후 재작성 작업
- 배포 누락: 판매 가능한 상품이 배포되지 않아 매출 기회 손실
- 거래처별 맞춤 배포 불가: 모든 거래처에 동일한 메일 발송

**자동화 목표**
- 신보 안내 이메일 수신 → AI 파싱 → 플랫폼 자동 등록
- 등록 즉시 전체 거래처 또는 관심 거래처에 알림톡/이메일 자동 발송
- 거래처별 과거 구매 이력 기반으로 관심 상품 필터링하여 맞춤 알림

---

### [3] 주문 접수 (수주서 발행)

**현재 방식**
- 거래처에서 이메일 또는 카카오톡으로 발주 수량 회신
- 담당자가 수주 내용을 ERP(이카운트)에 입력하여 수주서 발행
- 실리콘투(SLC)는 별도 워크시트 시스템으로 주문 소통 (아침마다 업데이트 메일 알림)

**주문 확정 기준**
- 수주서 발행 = 주문 확정
- **신보(예약 상품)**: 주문과 동시에 확정 (재고 이슈 없음)
- **구보(재고 상품)**: 발주처 재고 확인 후 주문 확정

**결제 시점**
- 고정 거래처: [8] 정산 시점에 세금계산서 발행
- 신규 거래처: 주문 확정 후 선결제
- 플랫폼 주문: 주문과 동시에 선결제 (취소 이슈 방지)

---

### [4] 발주 (발주서 발행)

**현재 방식**
- 수주 내용을 기반으로 매입처(유통사, 기획사, 일본 공급사)에 발주
- 발주서 양식은 거래처마다 상이
  - 웬즈데이 자체 발주서(엑셀) 작성 후 회신
  - 메이크스타: 자체 발주 요청서 양식 사용
  - 스타원코퍼레이션: 별도 발주서 없이 수량만 메일로 전달
  - 실리콘투: 워크시트 시스템 직접 입력

---

### [5] 구매 (매입)

**현재 방식 - 일본 수입의 경우**

```
인보이스 수신
    ↓ 익일 해외송금 (외국환거래계산서 발행)
    ↓ D+0 출항
    ↓ D+1 B/L 입수
    ↓ D+3 부산 입항
    ↓ 통관 + 육로 배송
    ↓ D+4~5 파주 물류창고 입고
```

**패킹리스트 처리**
- 동일 품목이 박스 단위로 중복 나열 → GPT로 품목별 합산 재분류
- JAN 코드 → 품번 매핑 필요 (일본 공급사가 엑셀 우측열에 수기 기재)
- 잡지류: 품번 없음, JAN 코드(한국 880코드 또는 ISBN 체계) 그대로 사용

---

### [6] 판매 (거래명세서 발행)

**현재 방식**
- 출고 시 거래명세서 발행 (ERP)
- 가격 체계:
  - **도서**: 정가 × 공급율 (예: 정가 10,000원 × 70% = 7,000원)
  - **음반**: 공장출고가 기준 → 거래처별 개별 가격 안내 필요 (단순 공급율 적용 불가)
  - **MD/굿즈**: 소매가 공급 (사운드웨이브 단독 MD의 경우, 매입가 = 소매가의 95%)

---

### [7] 출고

- 파주 물류창고에서 거래처로 출고
- 입출고현황 엑셀로 재고·주문·미입고·과입고 대조 관리

---

### [8] 정산

- 고정 거래처: 월말 또는 약정 시점에 세금계산서 발행 후 정산
- 해외 거래처: 외국환 거래 기반 송금 정산

## 업무 채널 현황

| 업무 | 현재 채널 | 자동화 가능성 |
|------|----------|--------------|
| 신보 안내 수신 | 이메일 (주), 카카오톡 (부) | ★★★ 이메일 파싱 자동화 |
| 신보 배포 | 이메일 수동 작성 | ★★★ 완전 자동화 목표 |
| 발주 접수 | 이메일, 카카오톡, 워크시트 | ★★★ 플랫폼 단일화 |
| 수주서 발행 | ERP(이카운트) 수동 입력 | ★★ ERP API 연동 or 자체 수주 관리 |
| 패킹리스트 처리 | GPT 활용 수동 처리 | ★★★ 자동 파싱 파이프라인 |
| 재고 관리 | 엑셀 (입출고현황) | ★★ 플랫폼 내 재고 모듈 |
| 정산 | ERP + 수동 확인 | ★ 1차 범위 외 |
| 일본 수입 물류 | 수동 (B/L, 통관 등) | ★ 복잡도 높음, 2차 범위 |
