# 개발 로드맵

## 비즈니스 초점 (1차)

초기 MVP는 **이미 계약·수기 관리 중인 거래처만**을 바이어로 두기보다, **중·소 규모 잠재 바이어**가 **검색(구글 등) → 상품 페이지 유입 → 이메일 인증 가입 → 주문 접수**까지 도달하는 경로를 검증하는 것을 우선한다. **견적·세부 발주·확정**은 1차에서 시스템 전부를 자동화하지 않고 **관리자 수기 프로세스**와 병행한다(상세는 [핵심 업무 프로세스](../business/process-flow.md)의 잠재고객 여정 참고).

---

## 전제: 데이터 준비 (개발 착수 전)

플랫폼 개발에 앞서 마스터 데이터 정비가 필요하다. **신규 유입 바이어는 자가 가입**으로 확장되므로, 아래 항목은 **내부 운영·가격 정책·기존 거래처**와의 정합을 위해 유지한다.

| 작업 | 담당 | 기한 |
|------|------|------|
| 거래처 마스터 정리 (코드, 정식명, 담당자, 결제유형) | 손정욱 대표 | 개발 착수 전 |
| 공급사 이메일 전체 목록 정리 | 손정욱 대표 | 개발 착수 전 |
| 거래처별 공급가 그룹 정의 | 손정욱 대표 + 강주원 | 개발 착수 전 |
| 기존 상품 DB 카테고리 정규화 (샘플) | 강주원 | 개발 착수 전 |

## Phase 1 — MVP (유입·가입·주문 접수 + 운영 기반)

**목표**: 상품 정보를 플랫폼에 반영하고, **검색 유입에 대비한 공개 상품 경험**에서 **이메일 인증 가입 → 주문 접수**까지 연결. 운영자는 **어드민에서 주문을 확인**하고, 이후 견적·발주는 수기로 처리.

### 기능 목록

**어드민 (운영자, Next.js UI → Django REST)**
- [ ] 상품 수동 등록 / 편집 / 삭제 (draft → published, SEO 필드 포함)
- [ ] 카테고리 관리 (국내반/수입반/MD 등)
- [ ] 아티스트 마스터 관리
- [ ] 거래처 등록 및 가격 그룹 설정 (기존 거래처·정책용)
- [ ] 주문 현황 대시보드 (접수 / 확정 / 출고) — **최소 접수 확인·상태 표시**
- [ ] 재고 수동 업데이트

> **이메일 파싱**: 현재 구축하지 않음. 상품은 관리자 직접 입력. 파싱은 Phase 2 [자동화 전략](../technical/automation.md)으로 이월.

**바이어 포털 (잠재·신규 바이어 중심)**
- [ ] **비로그인** 상품 상세·탐색 (검색엔진 유입 1차 방문 대응)
- [ ] **이메일 인증 기반 회원 가입** (전화번호 선택)
- [ ] 상품 카탈로그 탐색 (카테고리 / 아티스트 / 검색)
- [ ] 가격 표시 정책 (공개 범위는 비즈니스 규칙에 따름; 로그인 후 공개 등)
- [ ] 장바구니 → 주문 접수
- [ ] 주문 내역 조회

**기존 거래처 (병행 시)**
- [ ] 거래처 로그인 (승인 기반 또는 기존과 동일 정책)
- [ ] 거래처별 공급가 표시 (로그인 후 가격 공개)

**알림**
- [ ] 주문 접수 확인 알림 (바이어)
- [ ] (선택) 신보 등록 시 기존 거래처 대상 이메일·알림 — 세그먼트·운영 정책에 따라 단계 도입
- [ ] 주문 확정 / 출고 상태 알림 (거래처) — 후속 단계에서 강화

## Phase 2 — 자동화 고도화 + 글로벌 대응

**목표**: 전 카테고리 신보 자동화, 거래처별 맞춤 알림, 다국어 지원

### 기능 목록

**자동화 확장**
- [ ] 일본 도서·잡지 신보 이메일 파싱 (일본어)
- [ ] 패킹리스트 자동 합산 처리 모듈
- [ ] JAN 코드 ↔ 품번 자동 매핑
- [ ] 이메일 유형 자동 분류 (신보/재입고/구보전환/마감)
- [ ] 카카오톡 이미지 → 어드민 이미지 업로드 파싱

**맞춤화**
- [ ] 거래처별 관심 카테고리/아티스트 설정
- [ ] 관심 설정 기반 맞춤 신보 알림
- [ ] 카카오 알림톡 발송 연동 (이메일 외 채널 추가)

**글로벌 대응**
- [ ] 영어 다국어 지원 (i18n)
- [ ] 해외 결제 수단 (카드, 페이팔 등)
- [ ] 해외 거래처 온보딩 프로세스

**운영 도구**
- [ ] 재고 현황 대시보드 (미입고 / 과입고 파악)
- [ ] 거래처별 주문 실적 리포트
- [ ] 아티스트별 판매 실적 분석

### 기능 목록

- [ ] 성인 콘텐츠 전용 섹션 (인증 거래처 한정)

## 파일럿·검증 전략

Phase 1 이후 검증은 **단일 축**으로 고정하지 않는다.

**잠재 바이어(검색 유입) 검증**
- 특정 상품·키워드에서의 노출·클릭·유입(자연 검색 또는 소액 실험)
- 상세 페이지 → 가입 전환 → 주문 접수까지의 전환율
- 수기 견적·발주와 연결할 때의 운영 병목·품질

**기존 거래처 파일럿(선택)**
- 협조 가능한 소수 거래처에 한해 로그인·가격·주문 플로우를 병행 검증할 수 있다.
- 선정 기준 예: 이메일 발주를 규칙적으로 쓰는 곳, 디지털 툴 익숙도, 카테고리 다양성

**공통 목표**
- 주문·상품 데이터 흐름 실사용 검증
- (파싱 도입 후) 신보 파싱 정확도 검증
- 가격·노출 정책 오류 발견 및 수정
- 피드백 수렴 → 개선
